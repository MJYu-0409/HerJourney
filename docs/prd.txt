一、产品概述
1.1 产品定位
「她·伴」是面向围绝经期女性的微信小程序，以女性互助为核心理念，通过每日症状打卡、AI 智能问答、社区经验共享三大模块，陪伴用户完整经历更年期，并将个人数据自动转化为可读、可分享的经验帖与症状报告。

三、每日问卷模块（功能详细设计）
3.1 设计原则
分类展示症状：按身体系统分模块，降低认知负担
Tag 优先交互：症状以标签形式列出，点击选中变色，不强迫用户逐一判断
按需展开程度条：只有选中某症状 tag 后，该症状的严重程度滑块才出现，避免满屏滑块
默认不选即 0：未选中的症状视为「今日无此症状」，无需主动标记
分步填写：月经状态单独一步（Step 0），其余按模块在同一页滚动完成，减少页面切换
全程进度可见：顶部固定进度条；底部「完成」按钮始终可见

4.2 AI 回答逻辑（System Prompt 核心）
你是「她·伴」AI 伴侣，专注陪伴围绝经期女性。

回答结构（必须按此顺序）：
1. 【情绪共情】先认可用户感受（1-2句），不急于给建议
2. 【科普解释】用通俗语言解释症状成因
3. 【应对建议】给出 2-3 条可操作的日常建议
4. 【温暖结尾】一句鼓励或肯定

强制规则：
- 以下症状必须提示就医：异常出血、胸痛、剧烈头痛、骨折迹象
- 不做具体药物剂量推荐
- 回答长度控制在 200-300 字
- 末尾加注：*本回答参考更年期医学文献，仅供健康参考
4.3 快捷问题示例
「潮热总是突然来，怎么缓解？」
「睡不好和更年期有关系吗？」
「激素替代疗法安全吗，适合我吗？」

九、合规与注意事项
9.1 医疗内容合规边界
所有 AI 回答底部固定加注「本内容仅供健康参考，不构成医疗建议」
硬编码高危词触发就医提示：异常出血 / 胸痛 / 剧烈头痛 / 骨折迹象 → 弹窗提醒「请尽快就医」
产品定位为「健康科普与记录工具」，不使用「诊断」「治疗」等词汇
9.2 用户隐私
问卷和健康数据为用户私有，仅用于本人功能展示
经验帖发布前需用户主动确认，不自动公开任何健康数据
Demo 阶段不涉及真实微信用户数据，使用 Mock openid
9.3 微信开发者工具联调配置
开发阶段：微信开发者工具 → 详情 → 本地设置 → 勾选「不校验合法域名、web-view、TLS版本以及HTTPS证书」
后端启动：uvicorn main:app --reload --host 0.0.0.0 --port 8000
小程序 baseURL：开发环境 http://localhost:8000，统一在 src/config.ts 管理
图片/语音上传：调用 wx.uploadFile 上传至后端，后端保存至本地或对象存储返回 URL
